# 로그인/메인 HTML 목업 디자인

## 배경

초기 React 화면은 제품 방향을 보여주기에는 충분하지만, 발표/보고서/팀 구현 기준으로 쓰기에는 로그인 전후 화면 구조가 아직 명확하지 않다. 랜딩페이지보다 실제 웹앱 진입 흐름이 중요하므로, 별도 마케팅 랜딩은 만들지 않고 로그인 화면과 메인 앱 화면을 먼저 고정한다.

## 스타일 방향

- 기준 레퍼런스: getdesign.md의 Cal.com 계열 뉴트럴 UI
- 톤: 밝은 회색 배경, 흰색 패널, 검정 텍스트, 얇은 회색 border
- 강조색: 과도한 컬러 대신 국민대 느낌을 약하게 주는 딥 레드 포인트
- 카드 radius: 8px 이하
- 레이아웃: SaaS/학교 업무 도구처럼 조용하고 스캔하기 쉬운 앱형 화면

## 화면 범위

### 로그인 화면

- 프로젝트명과 한 줄 설명
- 이메일/비밀번호 입력 폼
- 데모 계정 안내
- RAG, 추천, 일정, LLM 기록 핵심 기능 요약
- 회원가입/비밀번호 재설정 진입 링크

### 메인 화면

- 좌측 사이드바: AI 질문, Mini Wiki, 추천, 일정, LLM 기록
- 상단바: 현재 사용자, 자료 업데이트 상태
- 중심 영역: RAG 질문 입력, 답변, 출처 목록
- 우측 영역: 추천 결과, D-day, LLM 기록 요약

## 구현 산출물

- `frontend/mockups/login.html`
- `frontend/mockups/main.html`
- `frontend/mockups/styles.css`

목업은 정적 HTML/CSS로 작성한다. React 구현 전에 브라우저에서 직접 열어 화면 구조를 확인하기 위한 산출물이며, 실제 인증/라우팅/API 호출은 포함하지 않는다.

## 비범위

- 실제 Supabase 인증 연동
- 실제 Gemini/RAG API 호출
- 반응형 세부 애니메이션
- 마케팅 랜딩페이지

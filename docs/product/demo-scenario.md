# 데모 시나리오

발표영상은 20-25분을 목표로 합니다.

## 1. 프로젝트 소개

- kmu-sw-navigator 소개
- 신입생의 정보 탐색 문제 설명
- RAG가 필요한 이유 설명

## 2. 온보딩과 메인 화면

- 데모 계정으로 로그인
- 소속, 학년, 입학년도/교과과정 설정
- AI가 선택지로 관심 영역을 질문
- 주요 탭 소개: 홈, 학업, 진로, 프로젝트, 일정, 기록
- 기록 탭에서 저장된 메모리 확인

## 3. 개인화 RAG 상담

질문:

```txt
AI학부 3학년인데 백엔드도 같이 준비해도 될까?
```

보여줄 것:

- 개인 메모리 근거
- 검색된 학교/학과 문서 출처
- 최신 웹 grounding 근거
- 답변 내용
- 확실한 것/제안/확인 필요를 분리하는 정책

## 4. 프로젝트/진로 추천 기능

입력:

```txt
관심 분야: AI
목표: 취업
코딩 경험: 초급
학습 성향: 프로젝트
```

보여줄 것:

- 조건문/딕셔너리 기반 추천
- 추천 이유와 액션 리스트
- PRD 초안, 과제 주제 추천, MVP 줄이기 같은 간단한 프로젝트 코치 기능

## 5. 일정 관리

입력:

```txt
자료구조 과제 다음주 금요일 밤 11시 59분까지 제출해야 해
```

보여줄 것:

- Gemini JSON 추출 결과
- 저장 전 미리보기
- D-day 표시
- Google Calendar 내보내기

## 6. LLM 활용 기록

- Gemini API 사용 기록
- Codex/Superpowers 기반 설계, 계획, 구현, 검증 기록
- 팀 협업 기록

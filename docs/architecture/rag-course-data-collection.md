# RAG 과목/수강신청 데이터 수집 기준

이 문서는 KMU SW Navigator의 RAG 답변 품질을 높이기 위해 과목 데이터, 수강신청 팁, 과목 후기, 진로별 추천 기준을 어떻게 모을지 정리한다.

핵심 원칙은 공식 자료와 학생 경험 자료를 분리하는 것이다. 공식 자료는 사실 확인과 졸업요건 판단에 쓰고, 학생 경험 자료는 추천과 주의사항에만 사용한다.

## 데이터 분류

| 분류 | 예시 | 사용 목적 | 신뢰도 표시 |
| --- | --- | --- | --- |
| 공식 과목 데이터 | 교과과정 PDF, 학사요람, 학부 공지 | 과목명, 학점, 이수구분, 학년/학기, 트랙 연결 | 공식 |
| 공식 제도 데이터 | 졸업요건, 트랙 안내, K-StarTrack, SW중심대학 공지 | 졸업요건, 현장실습, 인증, 장학/프로그램 안내 | 공식 |
| 수강신청 팁 | 선배 경험, 팀원 의견, 재학생 조언 | 어떤 순서로 들으면 좋은지, 무엇을 조심할지 | 비공식 경험 |
| 과목 후기 | 난이도, 과제량, 팀플 여부, 필요한 사전 지식 | 개인화 추천과 경고 문구 | 비공식 경험 |
| 진로 연결 데이터 | AI/백엔드/보안/게임 등 분야별 추천 과목과 활동 | 관심 분야별 로드맵 추천 | 혼합 |

## 1. 공식 과목 데이터

공식 과목 데이터는 가능하면 공식 페이지와 PDF에서 수집한다. 팀원이 임의로 과목 정보를 만들지 않는다.

수집 항목:

```txt
과목명:
학부: 소프트웨어학부 / 인공지능학부
적용 교과과정 연도:
학년:
학기:
이수구분: 전공필수 / 전공선택 / 교양 / 기초교양 / 기타
학점:
선수 과목:
관련 트랙:
핵심 키워드:
공식 설명:
출처 URL:
확인일:
```

출처 후보:

- 국민대학교 소프트웨어융합대학 교과과정 페이지와 PDF
- 인공지능학부 교과과정 페이지와 PDF
- 학사요람
- 학부 공지사항
- SW중심대학 공식 프로그램 자료

## 2. 수강신청 팁

수강신청 팁은 공식 자료만으로 부족하므로 팀원과 재학생 경험을 모은다. 단, RAG에 넣을 때는 반드시 `비공식 경험`으로 표시한다.

수집 항목:

```txt
팁 제목:
관련 과목:
대상: 1학년 / 2학년 / AI 관심자 / 백엔드 관심자 / 졸업예정자 등
내용:
주의사항:
근거 유형: 공식 / 선배 경험 / 팀원 의견
작성자:
확인일:
```

예시:

```txt
팁 제목: AI 관심자는 수학 기초와 자료구조를 같이 챙기기
관련 과목: 선형대수, 자료구조, Python
대상: 인공지능학부 1학년
내용: AI 관련 과목을 빨리 듣고 싶어도 선형대수와 자료구조를 먼저 잡아두면 이후 머신러닝/딥러닝 과목을 이해하기 쉽다.
주의사항: 실제 개설 학기와 선수 과목은 수강신청 전에 공식 시간표로 확인해야 한다.
근거 유형: 선배 경험
작성자: 팀원 A
확인일: 2026-05-25
```

## 3. 과목 후기

과목 후기는 추천 품질에 도움이 되지만 민감한 교수 평가나 개인정보는 넣지 않는다. 과목 자체의 학습 경험 중심으로만 정리한다.

수집 항목:

```txt
과목명:
들은 학기:
난이도: 낮음 / 보통 / 높음
과제량: 적음 / 보통 / 많음
팀플 여부:
시험 방식:
도움 되는 사전 지식:
추천 대상:
비추천 대상:
한 줄 후기:
주의사항:
근거 유형: 선배 경험 / 팀원 의견
확인일:
```

금지 항목:

- 교수 개인에 대한 평가
- 실명, 학번, 전화번호, 카카오톡 ID 같은 개인정보
- 특정 학생이나 조교를 식별할 수 있는 내용
- 확인되지 않은 소문

## 4. 진로/트랙 연결 데이터

과목 추천이 실제 서비스처럼 보이려면 “관심 분야 -> 추천 과목 -> 추천 활동 -> 프로젝트 예시” 연결이 필요하다.

수집 항목:

```txt
관심 분야: AI / 백엔드 / 프론트엔드 / 보안 / 게임 / 데이터 / 임베디드 / 창업
추천 과목:
추천 순서:
같이 하면 좋은 활동:
관련 프로젝트 예시:
관련 동아리/소모임:
관련 공식 자료:
근거 유형: 공식 / 선배 경험 / 팀원 의견
확인일:
```

## 데이터 저장 위치

원본과 정제본을 분리한다.

```txt
data/inbox/   # 팀원이 준 원본, 초안, 전사본, 캡처 설명
data/raw/     # RAG에 넣기 좋게 정제한 Markdown
data/wiki/    # Mini LLM Wiki로 컴파일된 문서
```

운영 방식:

1. 팀원이 자료를 `data/inbox/` 형식으로 제공한다.
2. 개인정보와 출처를 확인한다.
3. `data/raw/` Markdown으로 정제한다.
4. `pnpm wiki:build`로 wiki 문서를 갱신한다.
5. `pnpm rag:ingest:dry`로 chunk 결과를 확인한다.
6. live DB에 넣을 때는 `pnpm rag:ingest` 또는 embedding 경로를 사용한다.

## 팀원에게 보낼 요청 문구

```txt
RAG 상담 품질을 높이기 위해 과목/수강신청 데이터를 모으고 있습니다.

공식 자료에서 과목명, 학점, 이수구분, 교과과정 연도, 트랙 정보는 확인할 수 있지만, 실제 수강 순서, 과목 난이도, 과제량, 사전 지식, 진로별 추천 순서는 공개 자료만으로 부족합니다.

아래 내용을 가능한 범위에서 채워주세요.

1. 과목별 수강 후기
2. 수강신청 팁
3. AI/백엔드/보안/게임/데이터 등 진로별 추천 과목 순서
4. 같이 하면 좋은 프로젝트/동아리/대회
5. 주의해야 할 점

주의:
- 교수 개인 평가는 쓰지 말아 주세요.
- 실명, 학번, 연락처 같은 개인정보는 제외해 주세요.
- 공식 자료인지, 선배 경험인지, 팀원 의견인지 구분해 주세요.
```

## RAG 답변에서의 사용 원칙

- 졸업요건, 학점, 이수구분은 공식 자료만 근거로 답한다.
- 수강 난이도, 과제량, 수강 순서는 비공식 경험 자료라고 표시한다.
- 비공식 자료가 공식 자료와 충돌하면 공식 자료를 우선한다.
- 사용자에게는 “공식 기준”과 “선배 경험 기준”을 구분해서 보여준다.

# RAG 평가 기준

## 평가 질문

1. AI에 관심 있으면 어떤 트랙을 보면 좋을까?
2. 신입생이 수강신청 전에 뭘 확인해야 해?
3. 개발과 운동에 관심 있으면 어떤 활동이 좋을까?
4. 학교 시스템을 처음 쓰는 신입생이 알아야 할 것은?

실제 제출 리허설에서는 위 질문을 그대로 쓰되, 자동 평가 명령은 현재 RAG 자료에 직접 매칭되는 다음 질문 세트를 사용한다.

```powershell
pnpm rag:evaluate
```

자동 평가는 Gemini나 Supabase live key 없이 `data/raw`, `data/wiki`를 로컬 검색해 확인한다. 각 질문은 내부 출처가 1개 이상 잡히는지, 기대 category가 포함되는지, 근거 문서에 필요한 핵심어가 들어 있는지를 검사한다. 범위 밖 질문도 1개 넣어 내부 자료가 없는 주제를 억지로 근거 처리하지 않는지 확인한다.

## 평가 항목

| 항목 | 설명 |
| --- | --- |
| 출처 표시 | 답변에 근거 문서가 함께 표시되는가 |
| 근거 적합성 | 출처 내용이 답변을 실제로 뒷받침하는가 |
| 근거 부족 처리 | 자료가 부족할 때 모른다고 말하는가 |
| 신입생 친화성 | 답변이 쉬운 말과 실행 가능한 안내로 구성되는가 |
| 최신성 표시 | 확인이 필요한 정보는 공지 확인 필요를 표시하는가 |

## 초기 목표

- 라이브 시연 질문 4개에 대해 출처가 1개 이상 표시됩니다.
- 근거가 부족한 질문에는 과장된 답변을 하지 않습니다.
- 발표에서 검색 흐름을 설명할 수 있습니다.

## 최근 로컬 평가 결과

2026-05-27 기준 `pnpm rag:evaluate` 결과:

```txt
RAG quality evaluation: passed=5 failed=0 total=5
```

통과한 항목은 AI/트랙, 수강신청, 개발/활동, 교학팀 문의, 범위 밖 질문의 내부 근거 없음 처리다. 이 명령은 최종 답변 품질 자체를 평가하지 않고, 답변 생성 전에 사용할 내부 근거 후보가 시연 질문에 맞게 잡히는지를 빠르게 확인하는 게이트다.

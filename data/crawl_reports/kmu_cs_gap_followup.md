# KMU SW Navigator 공식 데이터 보강 점검

점검일: 2026-05-25

## 결론

`https://cs.kookmin.ac.kr/`는 내부 페이지 전체 탐색 기준으로 RAG 초기 데이터셋을 만들 수 있는 상태다. 다만 일부 핵심 정보는 HTML 텍스트가 아니라 PDF 또는 이미지로 제공되어, 단순 크롤러만으로는 서비스 품질이 떨어진다.

이번 추가 점검으로 `인공지능학부 졸업요건`은 비어 있는 페이지가 아니라 게시판 상세 이미지 자료라는 점을 확인했다. 2022-2026학년도 입학생 졸업요건 게시물이 존재하며, 2023-2026 자료 이미지는 `wfile.kookmin.ac.kr`에서 직접 다운로드 가능하다.

## 현재 부족한 부분

1. 학생 개인 데이터
   - 학번/입학년도, 학부, 현재 학년, 이수 과목, 관심 분야, 목표 진로, 다전공/부전공/심화전공 여부가 필요하다.
   - 이 데이터 없이는 추천이 “공식 정보 요약” 수준에 머문다.

2. 실제 수강/과제 일정
   - 학교 공개 사이트에는 학생 개인 시간표와 과제 마감이 없다.
   - 내부 캘린더를 구현하려면 사용자가 직접 입력하거나 CSV/엑셀/텍스트 업로드로 받아야 한다.
   - Google Calendar OAuth 없이도 `tasks`, `calendar_events`, `course_schedules` 테이블로 충분히 구현 가능하다.

3. 이미지/PDF 전처리
   - 일부 과거 교과과정 PDF와 AI 졸업요건은 이미지 기반이다.
   - 현재 로컬에는 `tesseract`가 없어 완전 자동 OCR은 안 된다.
   - 단기 조치는 핵심 표만 수동 정제해서 seed JSON/Markdown으로 넣는 방식이 제일 빠르다.

4. 동아리/소모임 정형 데이터
   - 공식 사이트에는 행사/프로그램 공지는 많지만, 동아리별 소개/모집/활동 주기 정보는 약하다.
   - 팀원이 알고 있는 동아리명, 설명, 모집 시기, 추천 대상, 링크를 별도 제공해야 추천 품질이 오른다.

5. 공지 상세의 첨부파일 처리
   - 목록 크롤링만으로 제목/작성일은 확보된다.
   - 실제 신청 기간, 제출 서류, 대상 학년은 상세 본문/첨부파일까지 파싱해야 한다.

## 앱에 바로 넣을 데이터 우선순위

1. `official_documents`
   - 트랙소개
   - SW/AI 2025 교과과정
   - SW/AI 졸업요건
   - 장학안내
   - 연구실/랩실
   - SW중심대학 주요 사업

2. `notices`
   - 학사공지
   - 취업공지
   - 장학공지
   - 특강 및 행사
   - SW중심대학 공지

3. `student_profile`
   - 학부, 입학년도, 학년, 관심 분야, 진로 목표, 이수 과목, 선호 활동

4. `internal_calendar`
   - 과제명, 과목명, 마감일, 중요도, 상태
   - 시험/신청/행사 일정
   - 공지에서 추출한 학교 일정 후보

## 사용자가 지금 주면 좋은 데이터

- 본인 학부/학번 또는 입학년도/학년
- 현재 수강 중인 과목 목록
- 이미 들은 전공 과목 목록
- 관심 분야: 예를 들어 AI, 백엔드, 웹, 보안, 게임, 데이터, 임베디드
- 목표: 취업, 대학원, 창업, 공모전, 현장실습, 트랙 선택, 졸업요건 확인 등
- 개인 일정/과제: 텍스트, CSV, 엑셀 중 아무 형식
- 팀에서 알고 있는 동아리/소모임/학회 정보

## 다음 구현 제안

구글 캘린더는 제외하고 내부 일정으로 간다.

- `calendar_events`와 `tasks`를 Supabase에 추가
- 공지 제목/본문에서 날짜 표현을 파싱해 “일정 후보”로 표시
- 사용자가 승인하면 개인 일정으로 저장
- 상담 화면에서는 “내 일정 + 공식 공지 + 졸업요건 + 트랙/교과과정”을 함께 근거로 답변

이 구조가 실제 서비스에 가깝다. 외부 OAuth 없이도 사용자가 체감하는 기능은 충분히 나온다.
